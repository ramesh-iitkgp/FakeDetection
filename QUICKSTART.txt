// Quick reference for running the system

// ============================================
// QUICK START GUIDE
// ============================================

// 1. BACKEND SETUP (5 minutes)
cd backend
python3 -m venv venv
source venv/bin/activate  // Windows: venv\Scripts\activate
pip install -r requirements.txt
python app.py
// Server runs at http://localhost:5000

// 2. EXTENSION SETUP (3 minutes)
// Chrome:
chrome://extensions/
-> Enable "Developer mode"
-> Load unpacked
-> Select extension/ folder
-> Done!

// 3. TEST THE SYSTEM
// In extension popup:
-> Paste an image URL or upload a file
-> Click "Analyze"
-> View results!

// ============================================
// KEY ENDPOINTS
// ============================================

POST /api/v1/analyze/image
- Analyze single image
- Input: URL or file upload
- Output: Detection results with risk level
- Time: 2-5 seconds

POST /api/v1/analyze/video  
- Analyze video frames
- Input: URL or file, sample_frames (default: 10)
- Output: Frame-by-frame analysis
- Time: ~1-2 seconds per frame

POST /api/v1/analyze/batch
- Batch analyze multiple images
- Input: Multiple files
- Output: Results for each
- Limit: 50 files

GET /health
- Health check
- Output: Server status
- Use for monitoring

// ============================================
// DETECTION METHODS (How It Works)
// ============================================

1. DEEPFAKE DETECTION (Ensemble)
   ├─ Face Detection (MediaPipe)
   ├─ Frequency Analysis (FFT)
   ├─ Texture Analysis (Laplacian)
   ├─ Eye Consistency Check
   └─ Blend Boundary Detection
   
2. METADATA ANALYSIS
   ├─ EXIF Data Validation
   ├─ Compression Detection (JPEG artifacts)
   ├─ Noise Consistency
   └─ Color Distribution Analysis

3. RISK SCORING
   ├─ Combine all indicators
   ├─ Weight by reliability
   └─ Output: Risk level (LOW/MEDIUM/HIGH/CRITICAL)

// ============================================
// TROUBLESHOOTING
// ============================================

Backend won't start:
→ Check Python version: python3 --version
→ Activate venv: source venv/bin/activate
→ Reinstall deps: pip install -r requirements.txt

Extension not loading images:
→ Verify backend is running: curl http://localhost:5000/health
→ Check API endpoint in Settings
→ Try F12 Console for errors

Slow analysis:
→ Check GPU: nvidia-smi
→ Restart Python interpreter
→ Use CPU-only mode temporarily
→ Reduce image resolution

Out of memory:
→ Process smaller files
→ Restart backend server
→ Reduce batch size
→ Increase system RAM/swap

// ============================================
// DEVELOPMENT & CUSTOMIZATION
// ============================================

Add Custom Detection Model:
1. Create model file in backend/models/
2. Register in utils/detector.py
3. Use in analysis pipeline
4. Test with sample images

Train on Custom Dataset:
1. Prepare annotated dataset
2. Create training script
3. Fine-tune base model
4. Validate accuracy
5. Deploy new weights

Modify Risk Scoring:
1. Edit calculate_overall_risk() in app.py
2. Adjust weights for each model
3. Change threshold values
4. Test with known samples

Deploy to Production:
1. Set FLASK_ENV=production
2. Use gunicorn instead of Flask dev
3. Configure HTTPS/SSL
4. Set up reverse proxy (nginx)
5. Enable monitoring/logging
6. Configure firewall
